# NASA ROBIN Rotor CFD Case (HiSA / OpenFOAM)

Reproducible CFD dataset for the NASA ROBIN rotor–fuselage configuration using OpenFOAM and HiSA.

[![DOI](https://zenodo.org/badge/1178676642.svg)](https://doi.org/10.5281/zenodo.18955571)

This repository provides a reproducible CFD simulation case for the NASA ROBIN helicopter configuration using OpenFOAM with the HiSA solver. The repository includes geometry files, mesh files, a complete OpenFOAM/HiSA simulation case, post-processing data and visualization files, and experimental reference data for validation. The dataset allows reproduction of the ROBIN rotor–fuselage interaction simulation and comparison with experimental measurements reported in NASA technical reports.

1. Geometry definition (01_geometry)  
2. Mesh generation (02_mesh)  
3. OpenFOAM / HiSA simulation case (03_case)  
4. Simulation execution  
5. Post-processing and validation (04_post)

# 1 Geometry

Directory: `01_geometry`

Contains STEP files for the ROBIN fuselage, pylon, and rotor blade geometry. The rotor blade geometry is based on a NACA0012 airfoil with −8° twist and rotor radius 0.86R. The fuselage and pylon geometry were generated using [stepROBIN](https://github.com/bosung-gotocloud/gotocfd/tree/main/stepROBIN)

---

# 2 Mesh

Directory: `02_mesh`

The mesh is generated using SALOME and exported to OpenFOAM format. The original SALOME project files (*.hdf) are large (~1.4 GB) and are not included in this repository. They can be downloaded from Zenodo:

DOI: https://doi.org/10.5281/zenodo.18955571

After downloading, place the .hdf files in this directory and follow the mesh export workflow. After merging the mesh, copy it into the simulation case:

```
cp -r robinmesh/constant/polyMesh ../03_case/case/constant/
```

# 3 Simulation Case (HiSA)

Directory: `03_case/case`

This directory contains a complete OpenFOAM case that can be directly executed using HiSA solver.

Structure:
```
case
├─ 0
├─ constant
│   ├─ dynamicMeshDict
│   └─ polyMesh
└─ system
```
## Initial Conditions

Initial conditions are defined in `0/`.  
Freestream conditions are defined in `0/include/freeStreamConditions`.

## Rotor Motion

Rotor motion is defined in `constant/dynamicMeshDict`. The multibody motion solver implementation can be found at: https://github.com/bosung-gotocloud/gotocfd/tree/main/multiMotionSolver

---

# Parallel Decomposition and AMI Handling

HiSA requires AMI patches to remain on a single processor during domain decomposition. This is handled using faceSets generated by topoSet.

Generate AMI face sets:
```
cd case  
topoSet
```
Decompose the domain for parallel simulation:
```
decomposePar -force
```
Run simulation (example with 64 cores):
```
mpirun -np 64 hisa -parallel
```
---

# Numerical Schemes

## Time Integration

Defined in `system/fvSchemes`. Dual-time stepping is used:
```
ddtSchemes { default bounded dualTime rPseudoDeltaT backward; }
```
## Linear Solver

Defined in `system/fvSolution`. The flow solver uses GMRES with LUSGS preconditioning. Pseudo-time iteration parameters include:
```
nPseudoCorr     10  
nPseudoCorrMin  8  
pseudoTol       1e-3  
```
Local time stepping is enabled.

---

# Simulation Control

Defined in `system/controlDict`.

Simulation parameters:

- Total revolutions      : 10  
- Time per revolution    : 0.02972 s  
- Steps per revolution   : 200  
- Total simulation time  : 0.2972 s  
- Time step: deltaT = 0.0001486  

Solution output is written every 10 time steps.

---

# Probe Locations

Pressure probes are defined using pointProbes in controlDict. These probes correspond to measurement locations used for comparison with experimental data. Recorded fields include: p, U, rho, rhoE.

# Post-Processing

Directory: `04_post`

This directory contains experimental reference data, processed CFD results, and ParaView state files.

Experimental data are taken from:

**Mineck, R. E., and Gorton, S. A.**, *"Steady and Periodic Pressure Measurements on a Generic Helicopter Fuselage Model in the Presence of a Rotor,"* NASA/TM-2000-210286, 2000.

---

# ParaView Visualization

ParaView state files (*.pvsm) are provided for visualization. Examples: centerline.pvsm, streamline.pvsm, vortex-core.pvsm, vorticity.pvsm, q.pvsm.

To reproduce the visualization:

1. Open ParaView  
2. Load a .pvsm file  
3. When prompted, select the simulation directory: 03_case/case
